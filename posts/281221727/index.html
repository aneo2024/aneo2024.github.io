循环神经网络（RNN）学习笔记（一）

一、基础篇：简单循环网络（SRN）核心原理

1.1 设计动机与核心创新

传统前馈神经网络（如MLP、CNN）存在两大局限：一是假设输入数据相互独立，无法捕捉序列数据的时序依赖（如文本中词的先后关联、视频中帧的连续关系）；二是输入输出维度固定，难以处理变长序列（如不同长度的句子、音频）。

SRN的核心创新是引入隐藏状态（Hidden State）和参数共享机制：

  • 隐藏状态 h_t：充当网络的“记忆单元”，编码从序列开始到当前时刻的所有历史信息，通过循环连接传递给下一个时间步。
  • 参数共享：所有时间步共用一套权重（U：输入→隐藏层，W：隐藏层→隐藏层，V：隐藏层→输出层），既减少参数数量避免过拟合，又能泛化到任意长度序列。

1.2 核心结构与数学表达

（1）时间展开视角

这种视角清晰展现了“记忆传递”（横向连接）和“当前信息处理”（纵向连接）的双重逻辑。

（2）关键公式

  • 隐藏状态更新（核心循环逻辑）：h_t = tanh(U·x_t + W·h_{t-1} + b)其中 tanh为激活函数，将输出压缩到[-1,1]区间，避免梯度爆炸初期风险。
  • 输出计算：y_t = Softmax(V·h_t + c)
    Softmax将输出转为概率分布，适用于分类任务（如情感分析、序列标注）。

1.3 SRN的典型结构类型

根据输入输出序列长度，SRN有5类核心应用结构：

  • 一对一（One-to-One）：本质是全连接网络，无循环特性（如简单分类任务）。
  • 一对多（One-to-Many）：单输入→序列输出（如图像生成文字、音乐生成）。
  • 多对一（Many-to-One）：序列输入→单输出（如文本情感分析、视频分类）。
  • 多对多（等长）：序列输入→等长序列输出（如帧级视频分类、词性标注）。
  • 多对多（不等长）：Encoder-Decoder结构（Seq2Seq），适用于机器翻译、文本摘要等任务。

二、进阶篇：参数学习与长程依赖问题

2.1 核心训练算法：时间反向传播（BPTT）

（1）核心逻辑

BPTT是训练RNN的基础算法，核心是“沿时间轴反向传播误差，累积梯度更新参数”：

  1. 前向传播：按时间步计算每个 h_t和 y_t，记录中间结果。
  2. 反向传播：从最后一个时间步T开始，将误差 e_T（如交叉熵损失）反向传递到每个时间步。
  3. 梯度累积：每个时间步的参数梯度（对 U、W、V）累加，最终通过梯度下降更新参数。

（2）关键梯度计算

以权重 W的梯度为例，误差沿时间传递的核心公式为：
∇_W e_T = Σ（从t=1到T）[δ_t · ∂h_t/∂W]
其中 δ_t为 t时刻的误差项，∂h_t/∂W包含 h_t对 h_{t-1}的梯度连乘项（∂h_t/∂h_{t-1} = tanh’(·)·W）。

2.2 核心痛点：长程依赖与梯度问题

（1）梯度消失/爆炸的本质

BPTT中，梯度需通过 ∏（从τ=k到t-1）tanh’(·)·W连乘传递（k为早期时间步）：

  • 梯度消失：tanh’(·)最大值为0.25，连乘后梯度随时间步呈指数衰减，趋近于0。
  • 梯度爆炸：若权重 W的范数大于1，连乘后梯度急剧增大，超出参数更新范围。

（2）长程依赖的影响

当序列过长（如长文本、长时序数据），早期时间步的梯度几乎衰减为0，导致模型无法更新早期参数，无法捕捉长距离关联（如“我是程序员，……，我擅长用____”中“程序员”与“编程”的关联）。

三、优化篇：模型改进、应用与图结构拓展

3.1 RNN的核心改进模型

（1）长短期记忆网络（LSTM）：精细门控解决方案

LSTM通过“细胞状态+三门控机制”解决长程依赖，核心是让关键信息无衰减传递：

  • 细胞状态（C_t）：类似“信息高速公路”，仅通过线性交互传递，避免非线性变换导致的梯度衰减。
  • 三门控功能（由Sigmoid函数控制，输出0~1表示信息保留程度）：
    1. 遗忘门（f_t）：决定丢弃 C_{t-1}中的无用信息（如文本中的虚词）。
    2. 输入门（i_t）：筛选当前输入的关键信息，更新到细胞状态。
    3. 输出门（o_t）：控制细胞状态的哪些信息输出到 h_t（短期记忆）。
  • 核心更新公式：C_t = f_t·C_{t-1} + i_t·tanh(U·x_t + W·h_{t-1} + b)。

（2）门控循环单元（GRU）：简化高效变体

GRU合并LSTM的门控结构，参数更少、训练更快，性能接近LSTM：

  • 核心改进：取消独立细胞状态，仅保留隐藏状态 h_t；将遗忘门+输入门合并为更新门（z_t），新增重置门（r_t）。
  • 关键逻辑：
    1. 更新门（z_t）：权衡保留旧状态（h_{t-1}）和接纳新信息（候选状态 h̃_t）。
    2. 重置门（r_t）：控制旧状态对候选状态的影响（r_t→0时忽略旧状态）。
  • 核心更新公式：h_t = (1-z_t)·h_{t-1} + z_t·tanh(U·x_t + W·(r_t·h_{t-1}) + b)。

（3）残差RNN：简单有效补充

如之前讨论的改进方法，通过 h_t = h_{t-1} + tanh(U·x_t + b)的线性残差连接，让梯度传递系数为1，避免梯度衰减，结构简单易实现。

（4）模型对比总结

模型 核心机制 长程依赖能力 优势 适用场景
SRN 无门控，直接Tanh变换 弱 结构简单、计算快 短序列、简单任务
LSTM 细胞状态+三门控 强 稳健、长期记忆可靠 长序列、复杂任务（翻译）
GRU 更新门+重置门 强 参数少、训练高效 中等长度序列、资源受限场景

3.2 RNN的核心应用场景

  • 自然语言处理（NLP）：机器翻译（Seq2Seq）、情感分析（多对一）、文本生成（一对多）。
  • 语音处理：语音识别（时序→文本）、语音合成（文本→时序）。
  • 时间序列预测：股价预测、气象数据预测（捕捉时序趋势）。
  • 图像处理：图像生成文字（看图说话）、帧级视频分类（等长多对多）。

3.3 图结构拓展：RNN与GNN的区别

GNN（图神经网络）是RNN在图结构数据的拓展，但两者核心逻辑完全不同：

对比维度 RNN（循环神经网络） GNN（图神经网络）
核心处理数据 序列数据（有时间顺序） 图结构数据（节点+边连接）
“循环”本质 沿时间步循环（时序依赖） 沿消息传递轮次循环（空间依赖）
参数共享方式 所有时间步共享同一套参数 同层节点共享参数，跨层独立
计算顺序 严格顺序执行（t=1→T） 同轮次可并行计算
核心应用 文本、语音、时序预测 社交网络、分子结构、知识图谱

GNN的核心是“消息传递”：节点通过聚合邻居信息更新自身嵌入，k轮迭代可捕捉k跳邻居依赖，无需时间维度约束。

四、实验实战：输入序列优化（针对长程依赖）

4.1 问题背景

用RNN处理32×32×3彩色图像（总特征数3072）时，若按 time_step=3072、input_size=1逐像素输入，序列过长导致梯度消失，模型无法捕捉长距离像素关联。

4.2 优化方案（满足 time_step×input_size=3072）

核心思路：减少 time_step（缩短序列长度）、增大 input_size（保留空间关联），缓解梯度衰减。

（3）结果分析

优化方案提升准确率的核心原因：

  1. 序列长度从3072缩短到32，梯度连乘步数大幅减少，缓解梯度消失。
  2. 每个时间步输入一行像素（3×32=96特征），保留了行内像素的空间关联，帮助模型学习局部特征后传递全局关联。

五、核心总结

  1. RNN的核心价值是通过“隐藏状态+参数共享”处理序列数据，解决传统网络的时序依赖和变长序列问题。
  2. 长程依赖的本质是BPTT中的梯度连乘效应，LSTM/GRU通过门控机制、残差RNN通过线性连接分别从结构和简化角度解决该问题。
  3. 输入序列优化是缓解长程依赖的低成本方案，核心是在固定总特征数的前提下，平衡 time_step和 input_size，保留数据的内在关联（如图像的行/块结构）。
  4. RNN与GNN的本质区别在于“循环维度”（时间vs空间），分别适配序列数据和图结构数据。
